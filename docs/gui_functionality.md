## 2. 游戏场景清单

### 场景 S01: 启动空白状态
- 窗口打开，无案件加载
- 嫌疑人下拉框为空
- 聊天区为空
- 倒计时显示 "--"
- 存档菜单项禁用
- 证据面板为空

### 场景 S02: 通过 AdminDialog 生成新案件
1. 菜单"案件 > 生成新案件" → 打开 AdminDialog
2. 选择背景类型
3. 输入模型名称（可选）
4. 点击[生成] → 等待LLM生成
5. 点击[验证] → 验证通过
6. 点击[保存] → 关闭对话框，案件加载到主界面
7. 嫌疑人下拉框填充嫌疑人姓名
8. 证据面板显示证据列表
9. 倒计时显示初始值

### 场景 S03: 选择嫌疑人开始审讯
1. 点击嫌疑人下拉框 → 选择嫌疑人A
2. 嫌疑人信息区更新（姓名+压力条）
3. 倒计时开始
4. 状态变为"interrogating"

### 场景 S04: 切换嫌疑人
1. 审讯中点击嫌疑人下拉框 → 切换到嫌疑人B
2. 嫌疑人信息区更新为嫌疑人B的信息
3. 倒计时继续

### 场景 S05: 发送聊天消息（标准审讯）
1. 在输入框输入问题
2. 点击[发送]或按回车
3. 输入框清空，控件禁用（等待LLM回复）
4. 聊天区显示"你: {问题}"
5. LLM回复后显示"嫌疑人: {回答}"
6. 控件重新启用

### 场景 S06: 发送空消息
1. 输入框为空时点击[发送]
2. 无任何反应

### 场景 S07: 发送空白字符消息
1. 输入框输入空格/Tab
2. 点击[发送]
3. 无任何反应

### 场景 S08: 施压操作
1. 点击工具栏[施压]
2. 聊天区显示施压消息
3. 压力值+10
4. 压力条更新

### 场景 S09: 共情操作
1. 点击工具栏[共情]
2. 聊天区显示共情消息
3. 压力值-5（不低于0）
4. 压力条更新

### 场景 S10: 出示相关证据（增加压力）
1. 点击证据面板中的相关证据
2. 弹出确认对话框"确定要出示证据「xxx」吗？"
3. 点击[是]
4. 聊天区显示证据出示消息
5. 嫌疑人回复
6. 压力值+20（相关证据）
7. 压力条更新

### 场景 S11: 出示不相关证据
1. 点击证据面板中的不相关证据
2. 确认出示
3. 嫌疑人回复
4. 压力值无额外变化

### 场景 S12: 取消出示证据
1. 点击证据面板中的证据
2. 弹出确认对话框
3. 点击[否]
4. 无其他变化

### 场景 S13: 破案成功（嫌疑人崩溃）
1. 持续施压使压力达到100
2. 触发 secret_triggered（秘密泄露）
3. 状态变为"breakdown"
4. 弹出对话框"破案成功！真凶已经崩溃认罪。"
5. 可选"重新开始"或"返回主菜单"

### 场景 S14: 时间耗尽（案件失败）
1. 倒计时归零
2. 状态变为"verdict"
3. 弹出对话框"时间耗尽！律师介入，案件被迫终止。"
4. 可选"重新开始"或"返回主菜单"

### 场景 S15: 重新开始
1. 在结束对话框中点击[重新开始]
2. 案件重新加载
3. 聊天记录清空
4. 压力恢复初始值
5. 倒计时恢复

### 场景 S16: 返回主菜单
1. 在结束对话框中点击[返回主菜单]
2. 界面恢复空白状态（S01）
3. 存档菜单项禁用

### 场景 S17: 存档
1. 菜单"游戏 > 存档"
2. 弹出成功提示"游戏已保存！存档ID: xxx..."
3. 存档写入数据库

### 场景 S18: 读档（无存档）
1. 菜单"游戏 > 读档"
2. 弹出提示"没有找到任何存档。"

### 场景 S19: 读档（有存档）
1. 菜单"游戏 > 读档"
2. 弹出存档选择对话框
3. 选择一个存档
4. 点击[确定]或双击
5. 游戏状态恢复（嫌疑人、压力、聊天记录、倒计时）
6. 弹出"读档成功"提示

### 场景 S20: 读档（取消）
1. 菜单"游戏 > 读档"
2. 弹出存档选择对话框
3. 点击[取消]
4. 无变化

### 场景 S21: LLM设置 - 选择Provider
1. 菜单"设置 > LLM 设置"
2. 打开设置对话框
3. 选择不同Provider
4. Base URL自动更新
5. 模型列表自动更新

### 场景 S22: LLM设置 - 测试连接
1. 打开设置对话框
2. 配置Provider/API Key/Base URL/模型
3. 点击[测试连接]
4. 显示"连接成功"或错误信息

### 场景 S23: LLM设置 - 保存设置
1. 配置完成
2. 点击[保存]
3. 设置写入配置文件
4. API Key存入keyring
5. 对话框关闭
6. 引擎LLM客户端重新初始化

### 场景 S24: LLM设置 - 取消
1. 打开设置对话框
2. 修改配置
3. 点击[取消]
4. 配置未保存

### 场景 S25: 操作进行中禁止重复操作
1. 发送消息后（LLM回复中）
2. 输入框和按钮禁用
3. 再次点击施压/共情 → 无反应
4. LLM回复后恢复

### 场景 S26: 无案件时操作无响应
1. 未加载案件
2. 点击施压/共情 → 无反应
3. 发送消息 → 无反应
4. 嫌疑人下拉框为空

### 场景 S27: 完整审讯流程（端到端）
1. 生成案件
2. 选择嫌疑人A
3. 发送3-4个问题
4. 切换到嫌疑人B
5. 发送2-3个问题
6. 出示相关证据
7. 施压
8. 共情
9. 存档
10. 继续审讯直到崩溃或超时

### 场景 S28: 压力边界测试
1. 持续施压 → 压力不超过100
2. 持续共情 → 压力不低于0

### 场景 S29: 倒计时精度
1. 选择嫌疑人开始审讯
2. 观察倒计时每秒递减1
3. 倒计时归零触发结束

### 场景 S30: AdminDialog - 生成失败处理
1. 输入无效模型
2. 点击[生成]
3. 显示错误信息
4. 可重新尝试

## 3. 关键交互流程

```
启动 → [案件 > 生成新案件] → AdminDialog → 选择背景 → [生成] → [验证] → [保存]
  ↓
主界面加载案件 → 嫌疑人下拉框 → 选择嫌疑人 → 倒计时开始
  ↓
审讯循环: [聊天/施压/共情/出示证据] × N → 压力变化 → 倒计时
  ↓
结局: 崩溃(breakdown) 或 超时(verdict) → [重新开始/返回主菜单]
```
